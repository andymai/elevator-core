# Unity Elevator Demo

A 3D elevator simulation demo using the `elevator-core` library via C FFI (P/Invoke).

## Prerequisites

- **Rust toolchain** (stable, 1.88+)
- **Unity 2022.3+ or Unity 6** (any render pipeline — uses OnGUI and `CreatePrimitive`)

## Build

```bash
# From the repo root or this directory:
bash examples/unity-demo/build.sh
```

This compiles `elevator-ffi` in release mode and copies the native library to `Assets/Plugins/Native/`.

## Setup

1. Open `examples/unity-demo/` as a Unity project
2. Click **Elevator > Create Demo Scene** in the menu bar
3. Press **Play**

The editor script generates the scene programmatically (no committed `.unity` file), so it works across Unity versions.

## Project Structure

```
examples/unity-demo/
  Assets/
    Scripts/
      ElevatorNative.cs       P/Invoke bindings (structs + DllImport)
      ElevatorSimulation.cs   Managed wrapper (IDisposable, frame copy)
      ElevatorSimManager.cs   MonoBehaviour (step, auto-spawn, events)
      BuildingVisualizer.cs   3D procedural rendering (cubes, orthographic)
      ElevatorHUD.cs          OnGUI stats + controls
    Editor/
      CreateDemoScene.cs      MenuItem to generate the demo scene
    Plugins/Native/           Native library (built by build.sh)
    StreamingAssets/config/
      default.ron             Elevator configuration
```

## Architecture

The demo uses P/Invoke to call the `elevator-ffi` C ABI. `ElevatorSimulation` wraps the native handle as an `IDisposable`, copying frame data from native pointers into managed arrays each frame. `ElevatorSimManager` is the MonoBehaviour that drives the simulation loop, auto-spawns riders via a Poisson timer, and counts lifecycle events. `BuildingVisualizer` creates procedural cubes in an orthographic side-view that matches the Bevy/Godot demos' visual language.

The scene is generated by an editor script (`Elevator > Create Demo Scene`) rather than a committed `.unity` file, making it resilient across Unity versions.
